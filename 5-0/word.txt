５章　ネットワーク基本用語３

▫️ワイヤレス(WLAN) 無銭LANのこと。ケーブル配線による制約がなく電波の届く範囲で自由に利用することができる、などのメリットがあるが、便利な反面遮蔽物等により電波が届かないことや、電波干渉によってうまく接続ができない、というデメリットもある

　・WLC(Wireless LAN Controller) 集中管理型のWLANにおいて使用されるデバイスのこと。複数台のAPを一元管理することができる。
　・集中管理型AP AP自身は電波の送受信のみを行う。設定・制御はWLCがまとめて行う。
　・自律型AP IOSが動作し、AP単独でWLANを提供できる。各APでそれぞれ設定を行う。
　・CSMA／CA WLANにおけるコリジョン回避方法のこと。コリジョン検出機能は持ち合わせていないため、ランダムな時間を空けてからデータ送信を行う。　
　・SOHO(Small Office/Home Office) 小規模なオフィスや自宅を拠点とした環境向けのネットワーク構成を指す。リモートアクセスが重要なためSOHO環境ではVPNがよく利用される。
　・CSS(Content Service Switch) ネットワークスイッチや＊ロードバランサ＊の一種。主にトラフィック管理やコンテンツの配信を最適化するために使用される。＊ロードバランシング：ネットワーク内の複数のサーバやリソースにトラフィックを効率よく振り分け、特定のサーバーへの負荷集中を防ぐ。
　　⚪︎BSSID(Basic Service Set Indentifier) Wi-Fiネットワークにおいて特定のアクセスポイント(AP)を識別するための一意の識別子のこと。APに割り当てられたMACアドレスとほぼ同じもので、無銭LANネットワーク内で各APを区別するために使用される。SSIDとは違い、SSIDとはWi-Fiネットワーク自体の名前のこと。BSSは１つのAPと、そのAPに接続するデバイスのグループのことを指す。
　・ESS(Extended Service Set) Wi-Fiネットワークの一形態であり、主に大きなエリアや複数の建物をカバーするために用いられる。複数のAPと、それらを結ぶ同じSSIDを共有するネットワークのグループのこと。複数のBSSが同じSSIDを持ち、広範囲をカバーする。　　
　　⚪︎ESSID(Extended Service Set Identifier) Wi-Fiネットワークを識別するための名前のこと。SSIDとほぼ同じ意味で使われる。技術的にはESSIDは拡張サービスセット全体の識別子を指す。
　・ローミング ユーザが一度接続したネットワークに移動中も途切れずにアクセスできるようにするための仕組み。接続を維持する技術を指す。APのスムーズな切り替え。国内・国際ローミングがあり、海外に行くと、自身の通信事業者のネットワークが使用できないため、現地の通信事業者のネットワークを利用して通信を行う。通常は追加で料金が発生する。
　・CAPWAP(Control and Provisionning op Wireless Access Points) 無線APと無線コントローラに間で通信を標準化するためのプロトコル。これにより、無線ネットワークの管理と制御が簡素化される。無線ネットワークの効率的な管理と運用を実現するために、無線ネットワークの構成要素の間のコミュニケーションを統一する重要なプロトコルとなる。

▫️FHRP(First Hop Redundancy Protocol) デフォルトゲートウェイを冗長化するためのプロトコル。HSRP・GLBP・VRRPと、３つの種類があり総称してFHRPと呼ぶ

　・HSRP(Hot Standby Router Protocol) 
　・VRRP(Virtual Router Redundancy Protocol) 
　・GLBP(Gateway Load Balancing Protocol) 
　・アクティブルータ 複数のルータをスタンバイグループとして１台の仮想ルータとして見せかけているが、実際に転送処理を行っているのは１台のルータであり、そのルータのことを指す。
　・スタンバイルータ アクティブルータのバックアップとなるルータのこと。アクティブルータに障害が発生した際に、その役割をスタンバイルータが引き継ぐ。

▫️QoS(Quality of Service) ネットワーク上で提供するサービスの品質を保つための技術のこと。

　・輻輳管理 パケットに割り当てられた優先順位を基に送信の順番を決定することで、輻輳をコントロールすること。キューイングとスケジューリングに関しては、方式がいくつかに分かれている。
　・輻輳回避 キューがいっぱいになった際に、それ以降に届いたパケットは全て破棄してしまうこと。
　　⚪︎テールドロップ 輻輳回避の１種。名のとおり、尾の部分を全て捨ててしまうこと。
　・帯域制御 特定の基準をもとに使用できるデータ転送量や通信速度を制限できること。企業などのWAN環境でよく使われるものになる。
　　⚪︎シェーピング 主にデータを送信する側で実装するもので、見められた制限レートを超えるパケットは一旦キューに保持しておいて、送信可能になったタイミングで送信するものになる。遅延やジッタという通信の乱れや揺れの発生の原因にもなってしまう。
　　⚪︎ポリシング 主にネットワークの入り口部分で実装するもの。制限レートを超えるパケットは破棄される。損失が起きる原因ではあるが遅延やジッタは発生しない。また、直前の転送データ量が少ない場合、制限レートを超えても送信が許可されることがある。

▫️WAN(Wide Area Network) 広範囲に渡るネットワークのことを指す。具体的には、地域や国、大陸などにわたる広い地理的範囲をカバーする。異なるネットワークを接続し、データの通信や共有を可能にする。

　・VPN(Virtual Private Network) プライベートで安全な通信を実現するための技術のこと。VPNを利用すると、ユーザーやデバイスはリモートで安全にプライベートネットワークにアクセスを行うことができ、データの機密性と生合成を保つことが可能となる。
　・インターネットVPN インターネットを利用してプライベートなネットワークに安全に接続するための技術のこと。これを利用すると、インターネット経由で企業や家庭のプライベートネットワークに接続し、データの暗号化とセキュリティを確保できる。
　　⚪︎サイト間VPN 地理的に離れた異なる拠点間でのネットワーク接続を実現するためのVPN。各拠点に設置されたルータやファイヤーウォールがVPNゲートウェイとして機能しデータの暗号化と複合化を行う。
　　⚪︎リモートアクセスVPN 離れた地点から、内部ネットワークやリソースにアクセスが可能になる。

　・IPsec(Internet Protocol security) インターネット上でデータを安全に送受信するためのプロトコルスのこと。主な機能は、暗号化・認証・整合性保護となりさらに、２つのモードで機能する。①トランスポートモード(ペイロード部分のみを保護。エンドツーエンド通信の際に使用)②トンネルモード(全体のIPパケットを暗号化する)
　・GRE(Generic Routing Encapsulation) 異なるネットワークプロトコルをトンネリングするためのプロトコル。接続する際、パケットを他のプロトコルでカプセル化することで、２つのネットワーク間を安全にデータ転送が可能になる。主にVPNや仮想プライベートネットワークの構築の際に利用されるが、暗号化やセキュリティ機能はないため、通常IPsecと併用される。
　・クラウド(クラウドコンピューティング) インターネット経由でデータ保存、計算能力、アプリケーションなどのITリソースを必要に応じて利用できるサービスのこと。物理的なハードウェアやインフラを管理しなくて良い利点がある。
　　⚪︎SaaS(Software as a Service)：完成されたソフトウェアをインターネット経由で提供するサービス。　GmailやSalesforceなどが該当
　　⚪︎PaaS(Platform as a Service)：アプリケーション開発や実行環境を提供するプラットフォーム。開発者はアプリ開発などに集中でき、インフラや運用はクラウドプロバイダーが管理する。　GoogleやApp Engineなどが該当
　　⚪︎IaaS(Infrastructure as a Service)：仮装サーバーやストレージなどのインフラを提供。ユーザーは必要に応じてリソースを借り、OSやアプリのインストール、管理を自分で行こなう。　AWS,EC2、Microsoft Azure などが該当

▫️セキュリティ 情報やシステムを、不正アクセス・データ改ざん・漏洩・破壊などの脅威から保護することを指す。これは機密性・完全性・可用性の３つの要素(※CIAトライアド)を維持することが目的となる。適切なセキュリティ対策を講じることで、データや資産を安全に保ち、信頼性を確保する。
　　
　　※CIAトライアド ①機密性：情報を許可された人だけがアクセスできるようにし、不正アクセスを防ぐこと。暗号化やアクセス制御がこれに該当する ②完全性：情報が改ざんされていないこと、正確で信頼できる状態を保つこと。データの改変を防ぐ仕組みや検証機能が含まれる。 ③可用性：必要な時に情報やシステムにアクセスできる状態を維持すること。サーバの冗長化やバックアップ・DDoS攻撃対策が該当する。　これらの３要素はセキュリティ対策の基礎となる。
　・脆弱性 システムやソフトウェアに存在する、セキュリティ上の弱点や欠陥のこと。
　・脅威 ネットワークやシステムの正常な動作、セキュリティ、データの安全性を損なう可能性のある[要因]や[行為]を指す。
　・マルウェア 有害なソフトウェアのこと 
　　⚪︎ウイルス コンピュータやネットワークに悪影響を与えるプログラムの一種のこと。データ破壊・情報漏洩・システム機能停止などの被害をもたらす。対策には、アンチウイルスソフトなどが必要となる
　　⚪︎ワーム ネットワークを通じて自己複製し、他のコンピュータやデバイスに感染を広げる悪意のあるソフトウェアのこと。ウイルスと異なり、ファイルやプログラムに規制せず、自立して動作する。ネットワークの帯域を消費し、通信速度を遅延させたり、観戦したコンピュータの動作を遅くすることが挙げられる。
　　⚪︎トロイの木馬 見た目は無害なソフトウェアやファイルを装い、ユーザが自らインストールしたり実行することで、コンピュータやネットワークに不正アクセスや、悪意のある行動を引き起こすマルウェアの一種。トロイの木馬は、感染後にシステム内で「バックドアの設置」や「情報収集」・「システムの操作」などを行うことがある。実行ファイルやメールの添付ファイルなどに隠されていることが多いため、警戒が必要。
　　⚪︎ランサムウェア コンピュータやネットワーク上のファイルを暗号化し、ユーザがアクセスできないようにロックするマルウェアの一種。
　・スプーフィング 通信やデータの送信元を偽装して、信頼できると見せかける手法のこと。これにより、攻撃者は信頼されるサーバーやユーザーとして振る舞い、不正アクセスやデータの盗難、情報の改ざんなどを行う。IPスプーフィング・メールスプーフィング・ドメインスプーフィングなどがある。
　　⚪︎DoS攻撃(Denial of Service Attack) システムやネットワークに対して大量のリクエストやトラフィックを送信し、そのリソースを過負荷にすることで、サービスを停止させる攻撃のこと。「フラッド攻撃」「リソース消耗攻撃」などがある。
　　⚪︎DDoS攻撃(Distributed Denial of Service Attack) 複数のコンピュータやネットワーク機器から一斉に大量のトラフィックを送り込み、ターゲットとなるシステムやネットワークを過負荷にする攻撃のこと。サービスの低下やパフォーマンスの低下を引き起こす。「ボリュームベース攻撃」「プロトコル攻撃」「アプリケーション層攻撃」などがある。
　　⚪︎中間者攻撃 通常の通信に攻撃者が介入し、通信愛用を盗聴・改ざん・偽装する攻撃手法のこと。通信のセキュリティが不十分な場合や、暗号化が施されていない場合に危険となる。対策として通信の暗号化(SSL/TLSなど)や、承認メカニズムの強化が重要となる。
　・ソーシャルエンジニアリング 心理的手法を使って人々を欺き、機密情報やアクセス権を不正に取得する攻撃手法のこと。攻撃者は信頼を築いたり、感情や社会的な圧力を利用して、ターゲットから情報や権限を引き出す。フィッシングメールやなりすまし電話などがある。
　・ブルートフォースアタック パスワードや暗号化キーを解読するために、可能なすべての組み合わせを試す攻撃手法のこと。対策としては、強力なパスワードの使用(長く複雑なPW)やアカウントロックアウトの設定が推奨される。
　・WPA(Wi-Fi Protected Access) 無線ネットワークのセキュリティを強化するために設計されたプロトコル。WEPのセキュリティの弱点を担うために開発された。不正アクセスやデータの盗聴などからネットワークを保護する。 

　　⚪︎WPAパーソナル 家庭や小規模オフィスの無線ネットワークで使用されるセキュリティプロトコルのこと。認証方法はPSK(Pre-Shared Key)を使用TKIP(Temporal Key Integrity Protocol)を使用してデータを暗号化する。WPA-2パーソナルでは、さらに強力なAES(Advanced Encryption Standard)を使用する。WPAパーソナルは簡単に設定できる一方で、漏洩するとネットワークが脆弱になる可能性がある。セキュリティを確保するためには、強力で複雑なパスフレーズの使用が推奨される。

　　⚪︎WPAエンタープライズ 企業や大規模ネットワーク向けに設計された、無線ネットワークのセキュリティプロトコルのこと。RADIUS(Remote Authentication Dial-In User Service)サーバーを使用して、ユーザー認証を行う。これにより、各ユーザーが個別に認証されるため、ネットワークセキュリティが強化される。TKIPやAESを使用してデータを暗号化する。WPA-2エンタープライズでは、AESが標準で使用されるため、より強力な暗号化が提供される。各ユーザに対して、個別の認証情報が提供されるため、ユーザの管理やアクセス制御が容易になる。

　・サプリカント ネットワークセキュリティにおいて、ユーザやデバイスがネットワークに接続する際に、認証を行うためのソフトウェア・機能のこと。
　・オーセンティケータ ネットワークセキュリティにおいて、接続要求を受け入れたデバイスやサーバーで、認証プロセスを実行する役割を持つコンポーネントのこと。
ネットワークアクセスを制御し、セキュリティを確保するための重要な役割を果たす。
　・認証サーバ ネットワークアクセスやネットワークサービスへのユーザー認証を担当するサーバーのこと。セキュリティを確保し、適切なユーザーだけがアクセスできるようにするための重要なコンポーネントとなる。(※コンポーネント：システムやプロセスを構成する個々の部分や要素のことを指す。ソフトウェアコンポーネント・ハードウェアコンポーネント・ネットワークコンポーネントなどが例としてあり、それぞれが独立して動作しつつ、全体として強調して機能することが求められる)

　・多要素認証 ユーザーがシステムやサービスにアクセスする際に、複数の認証要素を組み合わせてセキュリティを強化する認証法のこと。「知的要素」「所持要素」「生体要素」などが一般的に利用される。多要素認証を導入することで、パスワードだけに依存せずユーザーの身元を確認するため、セキュリティが大幅に強化される。これにより、不正アクセスのリスクが低減しアカウントやデータの保護が向上する。

　　⚪︎知識要素 ： パスワードやPINコードなど、ユーザが知っている情報のこと
　　⚪︎所持要素 ： スマートフォン、セキュリティトークン、カードなどユーザーが所持している物理的なアイテムのこと
　　⚪︎生体要素 ： 指紋、顔認証、声紋などのユーザーの身体的な要素のこと
　・AAA（Authentication,Authorization,Accoutingの略）
　　⚪︎Authentication(認証) ： ユーザやデバイスの正当性を確認するプロセスのこと。例）ユーザー名とPWの確認
　　⚪︎Authorization(認可) ： 認証されたユーザーやデバイスに対して、どのリソースやサービスにアクセスできるかを決定するプロセスのこと。例）アクセス制御リストによる権限設定
　　⚪︎Accounting(アカウティング) ： ユーザーデバイスのアクティビティを記録紙、かんさや分析のために使用するプロセス。例）ログの記録やトラフィックの監視
　　　AAAは、ネットワークのセキュリティを強化し、リソースの適切な管理と監視を実現する。
　・RADIUS(Remote Authentication Dial-In User Service) ネットワーク認証・認可・アカウティングを提供するプロトコル。特にリモートアクセスや無線ネットワークで広く使用される。クライアントと認証サーバーとの間でメッセージを交換し、ネットワークアクセスのセキュリティを確保する
